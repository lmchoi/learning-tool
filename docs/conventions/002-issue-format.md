# Convention — Issue format

## Principle
Issues describe value, not tasks. A reader should immediately understand what
they'll be able to do once the issue is done — not what files will be created.

## Structure
- **Title** — short, outcome-focused ("Generate a practice question", not "Add question_gen.py")
- **Body** — opens with what you can do after this is done and why it matters
- **Acceptance criteria** — observable outcomes, not implementation steps

## What not to include
- File names or implementation details in the title
- Task lists that describe how to implement rather than what done looks like
- Vague criteria like "works correctly" or "is tested"
